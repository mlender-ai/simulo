### Idea 1: JSON 파싱 실패 근본 해결 — 응답 검증 계층 분리

**발견 근거**:
- Git 로그에서 지난 30일간 **7개의 JSON 파싱 관련 fix** 커밋 발견:
  - `fix: JSON parse failure when Claude appends trailing prose after closing brace`
  - `fix: flow analysis JSON parse failures from truncation`
  - `fix: comparison analysis JSON truncation and recovery`
  - `fix: reorder JSON schema so thinkAloud/issues always generated first`
- QA 리포트에서 **"max_tokens truncation"** 으로 인한 500 에러 명시
- `lib/response-parser.ts` 파일이 2회 변경되었으나 반복 에러 발생

**현재 문제**:
- Claude API 응답이 max_tokens 초과 시 JSON이 잘려 파싱 실패
- 후행 문구(trailing prose)가 JSON 뒤에 붙으면 JSON.parse() 실패
- 각 API 엔드포인트에서 개별적으로 에러 처리 → 일관성 없음
- 타입 검증 후 기본값 할당 로직이 컴포넌트에 산재되어 있음

**제안**:
1. **Zod 기반 응답 검증 계층** 신규 생성 (`lib/validation/claude-response.ts`)
   - Claude 응답 구조에 대한 단일 스키마 정의
   - 부분 파싱 실패 시 자동 복구 로직 (JSON 잘린 경우 `null` 기본값 할당)
   - 타입 안전성과 런타임 검증 통합

2. **Response Parser 표준화** (`lib/response-parser.ts` 리팩토링)
   - 모든 API 엔드포인트에서 공통 파서 사용
   - Trailing prose 자동 제거 정규식 적용
   - 파싱 실패 시 구조적 폴백 제공

3. **API 라우트 통합** (`app/api/analyze/route.ts`, `app/api/analyze-flow/route.ts`)
   - 응답 검증 계층을 경유하도록 수정
   - 일관된 에러 응답 형식 정의

**구현 난이도**: Medium
**예상 임팩트**: High
**영향 파일**: 
- `lib/response-parser.ts`
- `lib/validation/claude-response.ts` (신규)

### Idea 3: TypeScript 빌드 파이프라인 안정화 — E2E 테스트 타입 에러 완전 해결

**발견 근거**:
- **QA 리포트 CRITICAL**: TypeScript 모듈 해석 실패 (`@playwright/test` 타입 선언 누락)
  - 3개 E2E 테스트 파일 모두 타입 에러 (`TS2307`, `TS7031`)
- **빌드 상태**: 타입 체크 `❌ FAIL` (4/4 항목 중 2개 실패)
- 최근 commit `4fdfc62`에서 **"resolve TypeScript build failure for E2E/Playwright files"** 시도했으나 미해결
- `package.json`과 `tsconfig.json`의 불일치 → 로컬 vs CI 환경 차이

**현재 문제**:
- `playwright.config.ts`, `e2e/*.spec.ts`에서 Playwright 타입 미인식
- 함수 매개변수(`page`)에 암묵적 `any` 타입 (TS7031)
- `skipLibCheck` 설정으로 임시 회피하고 있으나 근본 해결 안됨
- IDE에서 자동완성 미지원 → 개발 생산성 저하

**제안**:
1. **의존성 명시적 설치**
   ```bash
   npm install --save-dev @playwright/test@latest @types/node
   npm install --save-dev @types/react @types/react-dom
   ```

2. **`tsconfig.json` 통합 설정** (E2E 포함)
   ```json
   {
     "compilerOptions": {
       "skipLibCheck": true,
       "moduleResolution": "node",
       "resolveJsonModule": true,
       "types": ["@playwright/test", "node", "jest"],
       "lib": ["ES2020", "DOM"]
     },
     "include": ["app/**/*", "components/**/*", "lib/**/*", "e2e/**/*"]
   }
   ```

3. **E2E 테스트 타입 보강**
   ```typescript
   // e2e/fixtures.ts (신규)
   import { test as base, expect } from '@playwright/test';
   import type { Page, BrowserContext } from '@playwright/test';
   
   export const test = base;
   export { expect };
   ```

4. **CI/CD 검증 추가** (GitHub Actions)
   ```yaml
   - name: Type Check (E2E)
     run: npm run type-check:e2e
   ```

5. **검증 명령어 추가** (`package.json`)
   ```json
   "validate": "npm run lint && npm run type-check && npm run type-check:e2e && npm run build"
   ```

**구현 난이도**: Low
**예상 임팩트**: High
**영향 파일**:
- `package.json`
- `tsconfig.json`
- `tsconfig.e2e.json` (기존)
- `playwright.config.ts`
- `e2e/*.spec.ts` (3개 파일)
- `.github/workflows/auto-qa.yml`

**예상 작업량**: 1-2일
