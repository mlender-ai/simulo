### Idea 1: 히트맵 Zone → AI 분석 계약 레이어 도입 (HeatmapContract Interface)

**발견 근거**:
- `HeatmapViewer.tsx`가 핫스팟 3위(3회 변경), 관련 fix 커밋 3건: `fix heatmap zone accuracy and prevent overlap`(e72e3cc), `improve heatmap zone accuracy via OCR coords`(cb4ebe4), `replace simultaneous heatmap zones with single-zone picker UI`(f353f91)
- 히트맵 zone 정확도 문제가 UI 렌더링·OCR 좌표·프롬프트 그리드 세 계층에 걸쳐 각각 따로 패치되고 있음

**현재 문제**:
- zone 좌표가 `HeatmapViewer` → `ocr-extract` API → `lib/prompts.ts` → `opusGenerator`로 넘어가는 동안 타입 계약 없이 raw 숫자가 전달됨
- 렌더링 좌표계(CSS px)와 AI 프롬프트 좌표계(비율 %)가 혼용되어 "zone overlap"·"zone 이동" 버그가 반복 발생
- 수정할 때마다 렌더러, OCR 핸들러, 프롬프트 세 파일을 동시에 건드려야 해 회귀 위험이 높음

**제안**:
1. `lib/heatmap/types.ts`에 `HeatmapZone` 공통 인터페이스를 정의하고, **좌표 단위를 비율(0~1) 단일 기준**으로 통일
   ```typescript
   // lib/heatmap/types.ts
   export interface HeatmapZone {
     id: string;
     label: string;
     x: number;      // 0~1 (비율)
     y: number;
     width: number;
     height: number;
     confidence: number; // OCR 신뢰도
   }
   ```
2. `lib/heatmap/adapter.ts` 도입 — CSS px ↔ 비율 변환, OCR 좌표 → Zone 변환을 단일 함수로 캡슐화
3. `HeatmapViewer`는 비율 기반 Zone만 소비, `prompts.ts`는 동일 객체를 그리드 프롬프트로 직렬화
4. `outputSchema.ts`에 Zone 스키마 검증 추가로 AI 응답 단에서도 계약 보장

**구현 난이도**: Medium
**예상 임팩트**: High
**영향 파일**:
- `components/HeatmapViewer.tsx`
- `app/api/ocr-extract/route.ts`
- `lib/prompts.ts`
- `app/api/analyze/outputSchema.ts`
- `lib/heatmap/types.ts` *(신규)*
- `lib/heatmap/adapter.ts` *(신규)*

**예상 작업량**: 2~3일

### Idea 2: 멀티-프로덕트 모드를 위한 분석 컨텍스트 프로파일 시스템

**발견 근거**:
- `feat: add product mode toggle (야핏무브 ↔ 일반 서비스)`(1011235) 커밋으로 프로덕트 분기가 추가됨
- 동시에 `severity filter`(f2a515f), `user-selectable variant count`(cb4ebe4), `타깃 유저 4060세대 기본값`(7b9cded) 등 분석 파라미터가 파편적으로 늘어나는 중
- `app/api/analyze/route.ts`(핫스팟 4위, 3회 변경) + 핸들러 5개(`figma`, `comparison`, `url`, `flow`, `image`)가 이미 분기 처리로 복잡도 증가 중

**현재 문제**:
- 프로덕트 모드 토글이 추가되었으나, **모드별 기본 파라미터**(타깃 유저, 심각도 임계값, variant 수, 프롬프트 톤)가 코드 곳곳에 하드코딩
- 새 서비스 유형이 추가될 때마다 `route.ts`와 `prompts.ts`를 동시에 수정해야 함
- 사용자 입장에서도 모드를 바꿨을 때 이전 분석 설정이 초기화되지 않아 혼선 발생 가능

**제안**:
1. `lib/profiles/` 디렉토리에 **분석 컨텍스트 프로파일** 정의
   ```typescript
   // lib/profiles/types.ts
   export interface AnalysisProfile {
     id: 'yafitmove' | 'general';
     label: string;
     defaultTargetUser: string;
     defaultSeverityThreshold: 'low' | 'medium' | 'high';
     defaultVariantCount: 1 | 2 | 3;
     promptTone: 'fitness' | 'neutral';
     benchmarkScores: Record<string, number>;
   }
   ```
2. `lib/profiles/presets.ts`에 각 모드 프리셋 선언, 추후 CLAUDE.md나 환경변수 기반 확장 가능
3. `InputSection.tsx`의 파라미터 UI가 현재 선택된 프로파일에서 기본값을 자동 로딩
4. `app/api/analyze/route.ts`는 `profileId`를 받아 프로파일 객체를 핸들러에 주입 — 핸들러별 분기 코드 제거
5. 대시보드(`app/dashboard/page.tsx`)의 벤치마크 비교값도 프로파일에서 참조해 모드 전환 시 자동 갱신

**구현 난이도**: Medium
**예상 임팩트**: High
**영향 파일**:
- `app/api/analyze/route.ts`
- `app/api/analyze/handlers/` *(figma, url, image, flow, comparison 전체)*
- `components/InputSection.tsx`
- `app/dashboard/page.tsx`
- `app/api/dashboard/insights/route.ts`
- `lib/profiles/types.ts` *(신규)*
- `lib/profiles/presets.ts` *(신규)*

**예상 작업량**: 3~4일

### Idea 3: ImprovementPanel 생성 상태 머신 + 낙관적 UI (사용자 대기 마찰 제거)

**발견 근거**:
- `ImprovementPanel.tsx`가 핫스팟 2위(4회 변경): `fix reanalyze button doing nothing`(fd360ff), `restore panel + 3-variant support + PNG save`(cb4ebe4), `replace improvement panel with screenshot panel`(d0ec8cd), `rewrite generator prompt for surgical fixes`(9e12145)
- `feat: export status state machine with 5-stage per-stage feedback`(11ccfcc) 커밋이 이미 Export에는 상태 머신을 도입했으나, 개선안 생성(generate-improvement) 플로우에는 적용되지 않음
- `fix: reanalyze button doing nothing` — 버튼 클릭 후 아무 피드백 없어 사용자가 재클릭 → 중복 요청 발생

**현재 문제**:
- 개선안 생성은 Claude API 호출 시간이 10~30초로 길지만, 진행 상태가 단순 스피너 1개로만 표시됨
- variant 1~3개를 선택해도 생성 진행률이 variant별로 분리 표시되지 않아 "몇 번째 시안을 만들고 있는지" 알 수 없음
- 재분석(reanalyze) 버튼이 이전 결과를 즉시 지우고 빈 화면을 보여줘 컨텍스트가 단절됨
- PNG 저장 성공/실패 피드백이 없어 사용자가 저장 완료 여부를 알 수 없음

**제안**:
1. `lib/improvement/stateMachine.ts` 도입 — Export Hub와 동일한 5단계 패턴 적용
   ```typescript
   type ImprovementStage =
     | 'idle'
     | 'analyzing'           // 원본 이슈 재파싱
     | 'generating:1'        // variant 1 생성 중
     | 'generating:2'        // variant 2 생성 중
     | 'generating:3'        // variant 3 생성 중
     | 'rendering'           // PNG 렌더링
     | 'done'
     | 'error';
   ```
2. **낙관적 UI**: 재분석 시 이전 결과를 skeleton 카드로 교체 (완전 삭제 대신) → 컨텍스트 유지
3. variant별 스트리밍 진행바 — Claude API 응답이 스트리밍 가능하다면 partial JSON으로 실시간 표시, 불가능하면 예상 소요시간 기반 가상 프로그레스바
4. PNG 저장 시 토스트 알림(`✅ Variant 1 저장 완료`) 추가
5. 에러 발생 시 이전 결과를 복원하는 rollback 로직 — `useRef`로 마지막 성공 상태 보존

**구현 난이도**: Medium
**예상 임팩트**: Medium
**영향 파일**:
- `components/ImprovementPanel.tsx`
- `app/api/generate-improvement/route.ts`
- `lib/improvement/opusGenerator.ts`
- `lib/improvement/stateMachine.ts` *(신규)*

**예상 작업량**: 2일
