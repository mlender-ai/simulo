## Idea 1: API 응답 파싱 레이어 표준화 및 자동 복구 시스템

**발견 근거**: 
- Git log에서 JSON 파싱 관련 fix 커밋 7개 연속 (3ca6211, c52252b, 4e5a58a, 7dc4bed 등)
- QA 리포트: "JSON parse failure when Claude appends trailing prose after closing brace"
- 핫스팟 파일: `lib/storage.ts` (2회 변경), `app/api/analyze/route.ts` (반복 수정)

**현재 문제**: 
- Claude API 응답이 JSON 이후에 추가 텍스트를 붙이면 즉시 실패
- 각 엔드포인트별로 서로 다른 파싱 로직 구현 (일관성 부족)
- 필드 누락/undefined 검사가 분산되어 있음 (695a63e, 3777183 등에서 반복 수정)
- 토큰 제한으로 인한 truncation 문제 미해결 (7dc4bed, 64d6774)

**제안**: 
1. **중앙집중식 응답 파싱 엔진** 구축
   - `lib/claude/response-parser.ts`: JSON 추출, 재시도, 부분 복구 로직
   - Trailing prose 자동 제거, JSON 객체 경계 감지
   - 필수 필드 검증 스키마 (Zod/io-ts) 통합
   
2. **자동 재시도 및 부분 응답 처리**
   - Truncation 감지 시 → 필드 우선순위 기반 축약 (think_aloud → issues → suggestions)
   - 실패 시 → 같은 프롬프트로 자동 재요청 (지수 백오프)
   
3. **타입 안전성 강화**
   - 응답 타입을 한 곳에 정의 → 모든 API에서 재사용
   - Prisma schema와 응답 타입 자동 동기화

**구현 난이도**: Medium
**예상 임팩트**: High
**영향 파일**: 
- `lib/claude.ts` (리팩토링)
- `lib/claude/response-parser.ts` (신규)
- `app/api/analyze/route.ts`, `app/api/analyze-flow/route.ts` (통합)
- `lib/storage.ts` (에러 처리 단순화)

**예상 작업량**: 2-3일

## Idea 3: TypeScript + E2E 테스트 빌드 파이프라인 자동화 및 로컬 검증

**발견 근거**: 
- QA 리포트: "타입 체크 ❌ FAIL", "빌드 ⚠️ FAIL"
- 에러: `@playwright/test` 모듈 미등록, E2E 파일의 암묵적 any 타입
- 핫스팟: `package.json` (7회), `.github/workflows/auto-qa-ideas.yml` (3회)
- Git log: TypeScript 에러 수정 커밋 4개 (4fdfc62, 64d6774 등)

**현재 문제**: 
- `playwright.config.ts`, `e2e/*.spec.ts`에서 TS2307 (모듈 해석 실패)
- `tsconfig.json`과 `playwright` 설정이 동기화되지 않음
- 로컬 개발 환경과 CI 환경 간 결과 불일치 가능성
- E2E 테스트 타입이 불안정하여 유효성 검증 불가

**제안**: 
1. **빌드 파이프라인 통합**
   - GitHub Actions 워크플로우에 단계별 검증 추가:
     ```yaml
     - npm run type-check
     - npm run type-check:e2e  # 별도 tsconfig.e2e.json
     - npm run lint
     - npm run test            # E2E 테스트
     - npm run build           # 최종 빌드
     ```
   - 각 단계 실패 시 명확한 피드백 제공

2. **E2E 테스트 기본 구조 표준화**
   - `e2e/fixtures.ts` (신규): 공통 타입, 인증 페이지 fixture
   - 모든 `*.spec.ts` 파일: 명시적 타입 선언 강제
   ```typescript
   import { test, expect } from '@playwright/test';
   
   test('description', async ({ page }) => {
     // page는 자동으로 Page 타입 추론
   });
   ```

3. **개발 환경 자동 검증**
   - `npm run validate` 명령어 강제 (pre-commit hook)
   - `.husky/pre-commit`: lint + type-check + build test 순차 실행
   - dev 서버 시작 전 tsconfig 검증 스크립트

**구현 난이도**: Medium
**예상 임팩트**: Medium
**영향 파일**: 
- `tsconfig.json`, `tsconfig.e2e.json` (생성/수정)
- `playwright.config.ts` (설정 통합)
- `.github/workflows/auto-qa-ideas.yml` (단계 추가)
- `e2e/fixtures.ts` (신규)
- `e2e/*.spec.ts` (타입 추가)
- `.husky/pre-commit` (신규)

**예상 작업량**: 1.5-2일
