### Idea 1: Claude API 응답 파싱 안정화 — 전담 모듈화

**발견 근거**: 
- Git log에서 반복되는 JSON 파싱 에러 수정
  - `fix: JSON parse failures from truncation` (97048ac)
  - `fix: JSON parse failure when Claude appends trailing prose` (4e5a58a)
  - `fix: ensure all required fields exist after Claude response parse` (c52252b)
- lib/claude.ts가 893줄로 가장 많이 변경된 파일 (19회)
- app/api/analyze/route.ts도 3회 변경 (응답 처리 로직)

**현재 문제**: 
Claude API 응답 처리 로직이 lib/claude.ts에 산재되어 있어, 프롬프트 변경 → 응답 형식 변경 → 파싱 로직 수정이 반복되는 악순환 발생. 토큰 제한, 텍스트 잘림, 추가 문구 등 엣지 케이스별로 별도 fix가 필요했음.

**제안**: 
Claude 응답 파싱을 전담하는 `lib/claude/response-parser.ts` 모듈 분리
```typescript
// lib/claude/response-parser.ts
export class ClaudeResponseParser {
  static parseAnalysisResult(text: string): AnalysisResult {
    // 1. 텍스트 정제 (trailing prose 제거)
    // 2. JSON 추출 (```json ``` 블록 또는 원본)
    // 3. JSON 파싱 + 검증
    // 4. 누락 필드 보충 (기본값)
    // 5. 배열 경계 체크
  }
  
  // 테스트 가능한 단위 메서드들
  private static sanitizeTrailingText(text: string)
  private static extractJsonBlock(text: string)
  private static validateRequiredFields(obj: any)
}
```
이를 통해:
- 응답 처리 로직을 **단일 파일**에 집중 → 변경 시 영향도 명확
- 각 엣지 케이스별 단위 테스트 작성 가능
- API 라우트(`app/api/analyze/route.ts`)는 호출만 담당

**구현 난이도**: Medium
**예상 임팩트**: High (57% fix 커밋 중 30%가 파싱 관련)
**영향 파일**: 
- lib/claude.ts (분해)
- app/api/analyze/route.ts (의존성 단순화)
- app/api/analyze-flow/route.ts (동일)
- lib/claude/response-parser.ts (신규)

**예상 작업량**: 1-2일

### Idea 2: TypeScript 빌드 실패 → 테스트 인프라 신뢰도 확보

**발견 근거**:
- 최신 QA 리포트: **빌드 FAIL, 타입 체크 FAIL**
  - `@playwright/test` 모듈 해석 실패 (TS2307)
  - E2E 테스트 파일 암묵적 any 타입 (TS7031)
- Playwright 인프라는 4개 커밋으로 최근 추가 (1e83547, 4d3fe8d)
- package-lock.json이 4회 변경 (의존성 불안정)

**현재 문제**: 
E2E 테스트 인프라는 구축했으나, TypeScript 빌드가 실패해 자동화된 QA 파이프라인이 신뢰할 수 없는 상태. 개발자가 로컬에서 `npm run validate` 실행 시 실패하면 CI/CD 신뢰도 하락. Auto QA + Idea Generator 에이전트가 정상 작동하지 않을 가능성.

**제안**: 
1. **package.json 정리**: `@playwright/test@latest` 명시적 추가 + @types/node 확인
2. **tsconfig.json 동기화**:
   ```json
   {
     "compilerOptions": {
       "skipLibCheck": true,
       "types": ["@playwright/test", "@types/node"],
       "moduleResolution": "node"
     }
   }
   ```
3. **E2E 테스트 타입 안정화**: 
   - `e2e/fixtures.ts` 생성 → 공통 타입 정의 (Page, BrowserContext)
   - 기존 `e2e/*.spec.ts` 모두 명시적 타입 선언
4. **CI/CD 검증 추가**:
   ```bash
   # .github/workflows/auto-qa-ideas.yml
   - run: npm run type-check  # 현재 누락
   - run: npm run build       # 현재 누락
   ```

**구현 난이도**: Low
**예상 임팩트**: High (빌드 실패 해결 → 자동화 파이프라인 신뢰도 확보)
**영향 파일**:
- package.json
- package-lock.json
- tsconfig.json
- playwright.config.ts
- e2e/fixtures.ts (신규)
- e2e/*.spec.ts (3개 파일)
- .github/workflows/auto-qa-ideas.yml

**예상 작업량**: 0.5-1일
