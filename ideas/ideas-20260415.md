### Idea 2: 저장소 용량 자동 관리 시스템 (Storage Quota Management)

**발견 근거**:
- 최근 fix 커밋 2개: `3210914` (IndexedDB 이미지 저장), `64d6774` (localStorage quota exceeded)
- `lib/storage.ts` 3회 수정 기록 → 저장소 로직이 불안정
- 현재는 에러 발생 후 수동으로 캐시 정리 (`npm run dev:clean`) 필요

**현재 문제**:
- heatmap overlay, flow analysis 결과 등 대용량 데이터가 localStorage에 누적
- 사용자가 여러 프로젝트를 분석할 때 quota 초과로 앱 크래시
- IndexedDB와 localStorage 용량 관리가 분리되어 있어 예측 불가능

**제안**:
```typescript
// lib/storage-manager.ts (신규)
class StorageManager {
  async saveWithQuotaCheck(key: string, data: any, priority: 'critical'|'temporary'): Promise<void>
  
  // 기능:
  // 1. 저장 전 용량 예측 (gzip 압축 추정)
  // 2. 초과 시 자동 LRU eviction (임시 데이터부터 삭제)
  // 3. critical 데이터는 압축 저장 후 필요시만 decompress
  // 4. 저장소 사용량 대시보드 (settings/storage-usage)
  
  getStorageStats(): { used: number, total: number, breakdown: {...} }
  clearTemporaryData(): void
  compressAnalysisResult(data: AnalysisResult): CompressedResult
}
```

**구현 난이도**: Medium
**예상 임팩트**: High (앱 크래시 방지, 사용자 체험 향상)
**영향 파일**:
- `lib/storage.ts` (리팩토링)
- `components/InputSection.tsx` (저장 전 용량 체크)
- `app/settings/page.tsx` (저장소 관리 UI)
- `app/api/analyze/route.ts`, `app/api/analyze-flow/route.ts`

**예상 작업량**: 2-3일

### Idea 3: E2E 테스트 기반 빌드 게이트 (TypeScript + Test Coverage 자동화)

**발견 근거**:
- 최신 QA 리포트: 빌드 ⚠️ FAIL, 타입 체크 ❌ FAIL
- 문제: `@playwright/test` 모듈 타입 누락, E2E 테스트 3개 파일 모두 암묵적 any 타입
- `package.json` 변경 7회, `package-lock.json` 6회 → 의존성 관리 불안정
- CI/CD (`auto-qa-ideas.yml`) 구성되어 있으나 빌드 실패 상황에서도 merge 가능

**현재 문제**:
- 개발자가 로컬에서 성공해도 CI에서 타입 체크 실패
- E2E 테스트가 자동 실행되지만 **타입 검증 없이** 런타임 에러 위험
- `tsconfig.json`과 `tsconfig.e2e.json` 동기화 불완전
- QA 에이전트(`qa-agent.ts`)가 실행되지만 **빌드 실패 시 배포 차단 없음**

**제안**:
```yaml
# .github/workflows/auto-qa-ideas.yml (개선)
jobs:
  validate:
    - npm run type-check
    - npm run type-check:e2e
    - npm run test          # Playwright
    - npm run build         # Next.js build
    
    # 빌드 실패 시 자동으로:
    - 원인 분석 (type-check 결과 파싱)
    - issue 자동 생성 (severity: critical)
    - PR 코멘트로 수정 제안
    - main 병합 차단
```

**구체적 개선**:
1. **TypeScript 통합**: `tsconfig.e2e.json`의 설정을 메인 `tsconfig.json`에 병합
   ```json
   {
     "compilerOptions": {
       "skipLibCheck": true,
       "types": ["@playwright/test", "node", "@types/react"],
       "moduleResolution": "bundler"
     }
   }
   ```

2. **E2E 테스트 기본 설정 제공**:
   ```typescript
   // e2e/base.setup.ts
   import { test as base } from '@playwright/test';
   
   export const test = base.extend({
     authenticatedContext: async ({ browser }, use) => { /* ... */ }
   });
   ```

3. **자동 타입 생성**: `postinstall` 스크립트에서 Playwright 타입 자동 생성

**구현 난이도**: Low
**예상 임팩트**: High (빌드 안정성 100%, 런타임 에러 사전 차단)
**영향 파일**:
- `.github/workflows/auto-qa-ideas.yml`
- `tsconfig.json`, `tsconfig.e2e.json`
- `package.json` (postinstall 스크립트)
- `e2e/*.spec.ts` (3개 파일)
- `playwright.config.ts`

**예상 작업량**: 1-2일
