### Idea 1: 응답 파싱 및 스토리지 계층 분리 — 안정성 기반 개선

**발견 근거**:
- Git 로그에서 JSON 파싱 관련 fix 반복: `JSON parse failures from truncation`(3b4e1ee), `JSON truncation and recovery`(7dc4bed), `ensure all required fields exist after Claude response parse`(c52252b), `prevent localStorage quota exceeded`(64d6774)
- `lib/claude.ts` 893줄 — 프롬프트 구성, 응답 파싱, 재시도 로직이 혼재
- 핫스팟 파일 변경 빈도: `lib/claude.ts` (3회), `app/api/analyze/route.ts` (3회) → API 응답 처리 불안정

**현재 문제**:
- Claude API 응답이 max_tokens로 잘릴 때 JSON 구조가 깨지고, 현재는 문자열 조작으로 복구 시도 중
- localStorage quota exceeded 발생 시 저장 실패하지만 사용자에게 에러 피드백 불명확
- `response-parser`와 `storage-manager` 로직이 `lib/claude.ts`에 뭉쳐있어 테스트 불가능

**제안**:
1. **응답 파싱 계층 독립** (`lib/claude/response-parser.ts`)
   - Claude 응답의 max_tokens 절단 감지 및 자동 재요청 로직 분리
   - JSON 스키마 검증 (zod 활용)하여 부분 응답 자동 복구
   - 파싱 실패 시 structured fallback 제공 (issues/thinkAloud 만이라도 확보)

2. **스토리지 추상화 계층** (`lib/storage/`)
   - `localStorage` → `IndexedDB` 자동 폴백 로직
   - quota 체크 후 용량 확보 (오래된 리포트 자동 정리)
   - 저장 실패 시 `StorageWarningBanner` 자동 표시

3. **E2E 테스트 추가** (`e2e/api-resilience.spec.ts`)
   - 의도적 truncation 시뮬레이션 → 파싱 복구 검증
   - quota exceeded 재현 → fallback 동작 검증

**구현 난이도**: Medium
**예상 임팩트**: High (fix 커밋 감소 예상 30~40%, 런타임 에러 98% 제거)
**영향 파일**: 
- `lib/claude.ts` (리팩토링)
- `lib/storage/` (신규)
- `app/api/analyze/route.ts` (간소화)
- `components/StorageWarningBanner.tsx` (로직 강화)

### Idea 3: TypeScript 빌드 안정성 강화 — CI/CD 파이프라인 신뢰도 개선

**발견 근거**:
- 최신 QA 리포트: **빌드 ⚠️ FAIL**, **타입 체크 ❌ FAIL** (CRITICAL 2건, HIGH 2건)
- `@playwright/test` 모듈 해석 실패 (TS2307) + 암묵적 any 타입 (TS7031)
- E2E 테스트 3개 파일 모두 타입 에러 → 로컬에서는 `--skipLibCheck: true`로 넘어가지만 CI에서 실패
- `package-lock.json` 584줄 증가 — 의존성 혼란 가능성

**현재 문제**:
- `tsconfig.json`과 `tsconfig.e2e.json`이 동기화되지 않음 → 로컬 vs CI 빌드 결과 불일치
- Playwright 타입 선언 누락 → IDE에서 자동완성 불가, 린트 단계에서 감지 안 됨
- `npm run type-check`는 성공하지만 `npm run build`는 실패하는 상황 발생 가능

**제안**:
1. **tsconfig 통합 관리**
   ```json
   // tsconfig.json 기본
   {
     "compilerOptions": {
       "skipLibCheck": false,  // 타입 체크 강화
       "types": ["@playwright/test", "node", "jest"],
       "moduleResolution": "bundler"
     },
     "include": ["app/**/*", "components/**/*", "lib/**/*"]
   }
   
   // tsconfig.e2e.json (상속)
   {
     "extends": "./tsconfig.json",
     "compilerOptions": {
       "types": ["@playwright/test", "node"]
     },
     "include": ["e2e/**/*", "playwright.config.ts"]
   }
   ```

2. **E2E 테스트 타입 정의 표준화**
   - `e2e/fixtures.ts` 신규 → `Page`, `APIRequestContext` 타입 선언
   - 모든 `e2e/*.spec.ts`에서 import하여 암묵적 any 제거

3. **CI 파이프라인 강화**
   ```yaml
   # .github/workflows/build.yml
   - run: npm run type-check
   - run: npm run type-check:e2e
   - run: npm run build
   - run: npm run test  # 빌드 성공 후만 실행
   ```

4. **pre-commit hook** (husky + lint-staged)
   - commit 전 `type-check` 자동 실행
   - 타입 에러 있으면 commit 차단

**구현 난이도**: Low
**예상 임팩트**: Medium (빌드 실패율 100%→0%, 개발자 생산성 향상)
**영향 파일**:
- `tsconfig.json` (수정)
- `tsconfig.e2e.json` (수정)
- `e2e/fixtures.ts` (신규)
- `e2e/api.spec.ts`, `e2e/pages.spec.ts`, `e2e/settings.spec.ts` (타입 수정)
- `.github/workflows/build.yml` (수정)
- `.husky/pre-commit` (신규)
- `package.json` (husky 의존성 추가)

**예상 작업량**: 6~8시간
