## Idea 1: TypeScript 빌드 파이프라인 안정화 (타입 검사 자동화)

**발견 근거**: 
- QA 리포트: 타입 체크 ❌ FAIL (TS2307, TS7031 에러)
- Playwright E2E 인프라 구축 완료했으나, `@playwright/test` 모듈 타입 선언 누락
- 최근 4f7186c, 1e83547 커밋에서 E2E 테스트 인프라 추가했으나, 빌드 검증 미흡
- `npm run type-check:e2e` 스크립트 존재하나 CI/CD 파이프라인에 미통합

**현재 문제**: 
- 빌드 단계에서 TS 타입 검사 실패 → 배포 불가
- IDE에서 `page`, `test` 등 Playwright API 자동완성 불가능
- 로컬 개발과 CI 환경 간 타입 검사 설정 불일치 (`tsconfig.json` vs `tsconfig.e2e.json`)
- E2E 테스트 3개 파일 모두 암묵적 `any` 타입으로 인한 런타임 리스크 증가

**제안**: 
1. **의존성 강제 설치**: `npm install --save-dev @playwright/test@latest @types/node` 추가 및 잠금
2. **tsconfig.json 통합**: `skipLibCheck: false`, `types: ["@playwright/test", "node"]` 명시적 선언
3. **타입 검사 강제화**: 
   - `npm run type-check` 에 E2E 타입 검사 통합 (`tsc -p tsconfig.e2e.json` 병렬 실행)
   - GitHub Actions에 `type-check` 단계 추가 (빌드 전 필수 실행)
4. **E2E 파일 타입 수정**: `test('...', async ({ page }: { page: Page })` 패턴 일괄 적용
5. **공통 타입 모듈 신규**: `types/playwright.ts` 생성 → 타입 정의 중앙화

**구현 난이도**: Low
**예상 임팩트**: High
**영향 파일**: 
- `package.json` (의존성 추가)
- `tsconfig.json`, `tsconfig.e2e.json` (타입 검사 설정)
- `e2e/*.spec.ts` (3개 파일 타입 선언 추가)
- `.github/workflows/*.yml` (CI 파이프라인 수정)
- `types/playwright.ts` (신규 생성)

**예상 작업량**: 4-6시간

## Idea 2: 핫스팟 컴포넌트 분해 및 모듈화 (대용량 파일 리팩토링)

**발견 근거**: 
- **핫스팟 파일 분석**:
  - `lib/claude.ts`: 893줄 (API 통신 + 프롬프트 생성 + 응답 파싱 혼재)
  - `components/InputSection.tsx`: 824줄 (파일 업로드 + 파라미터 설정 + 프리셋 선택 혼재)
  - `components/ReportTabs.tsx`: 811줄 (탭 레이아웃 + 분석 결과 렌더링 혼재)
- 7일간 가장 많이 수정된 파일 (19회 변경)
- 단일 책임 원칙(SRP) 위반으로 인한 테스트 작성 어려움, 변경 시 부작용 위험

**현재 문제**: 
- 한 파일에서 여러 책임 담당 → 변경 영향도 예측 어려움
- 테스트 작성 시 모든 로직을 mock해야 함 → E2E 테스트 작성 부담 증가
- 코드 재사용성 낮음 (다른 페이지에서 InputSection 로직만 필요해도 전체 import)
- 성능 최적화 기회 상실 (불필요한 리렌더링)

**제안**: 
1. **`lib/claude.ts` 분해**:
   ```
   lib/claude/
   ├── ai-client.ts      (Claude API 호출, exponential backoff 로직)
   ├── prompt-builder.ts (프롬프트 생성, i18n 통합)
   ├── response-parser.ts (이미 분리됨, import 최적화)
   └── index.ts          (export 통합)
   ```
2. **`components/InputSection.tsx` 분해**:
   ```
   components/input/
   ├── InputSection.tsx   (컨테이너, 상태 관리)
   ├── FileUploader.tsx   (파일 업로드 UI)
   ├── ParameterControl.tsx (파라미터 입력 UI)
   ├── PresetSelector.tsx (프리셋 선택 드롭다운)
   └── hooks/
       └── useInputState.ts (상태 로직)
   ```
3. **`components/ReportTabs.tsx` 분해**:
   ```
   components/report/
   ├── ReportTabs.tsx    (탭 컨테이너)
   └── tabs/
       ├── OverviewTab.tsx
       ├── DetailTab.tsx
       └── ComparisonTab.tsx
   ```
4. **공통 타입 통합**: `types/domain/` 디렉토리 신규 → 기능별 타입 정의

**구현 난이도**: Medium
**예상 임팩트**: High
**영향 파일**: 
- `lib/claude.ts` → 분해
- `components/InputSection.tsx` → 분해
- `components/ReportTabs.tsx` → 분해
- `lib/response-parser.ts` (import 경로 수정)
- `app/page.tsx`, `app/analyze/page.tsx` (import 경로 수정)
- `e2e/*.spec.ts` (테스트 시나리오 단순화 가능)

**예상 작업량**: 12-16시간

## Idea 3: E2E 테스트 기반 구축 + 자동화된 회귀 테스트 파이프라인

**발견 근거**: 
- Playwright E2E 테스트 인프라 최근 구축 (1e83547 커밋, 7일 전)
- 현재 3개 E2E 테스트 파일만 존재 (`api.spec.ts`, `pages.spec.ts`, `settings.spec.ts`)
- 최근 `exponential backoff retry` 추가 (770c8c0), 비교 분석 탭 추가 (5ffacb5), 공유/내보내기 기능 추가 (f9acd89) 등 주요 기능 변경이 많으나 **테스트 커버리지 불명시**
- QA 에이전트 구축 완료 (2386b95)했으나, **자동화된 회귀 테스트 미흡**

**현재 문제**: 
- 주요 기능 변경 시 수동 테스트에 의존
- 자동 리포트 생성 (매일) 중일 때, 실제 기능 변경으로 인한 **회귀 버그 탐지 미흡**
- E2E 테스트가 있어도 **사용자 플로우 중심 테스트 부재** (예: 파일 업로드 → 분석 실행 → 리포트 내보내기 전체 플로우)
- CI/CD에 E2E 테스트 자동 실행 통합 미흡

**제안**: 
1. **주요 사용자 플로우별 E2E 테스트 추가**:
   - `user-flow-complete-analysis.spec.ts`: 파일 업로드 → 분석 → 결과 조회 → 내보내기 (PNG/PDF/DOCX)
   - `user-flow-comparison.spec.ts`: 두 개 프로젝트 비교 분석 (비교 탭 사용)
   - `user-flow-history-share.spec.ts`: 분석 히스토리 조회 → 공유 링크 생성 → 다른 브라우저에서 접근
   - `api-resilience.spec.ts`: Claude API 재시도 로직 (exponential backoff) 검증

2. **테스트 픽스처 표준화**:
   ```typescript
   // e2e/fixtures.ts
   - authenticatedPage (로그인 상태 Page)
   - sampleImageFile (테스트 이미지)
   - cleanupDb (테스트 후 DB 정리)
   ```

3. **CI/CD 통합**:
   - GitHub Actions: `test` 단계 추가 (빌드 후, 배포 전)
   - E2E 실패 시 스크린샷/비디오 자동 수집
   - 슬랙/이메일 알림 (실패 시)

4. **자동화된 회귀 테스트 스케줄**:
   - 매일 야간 (00:00 UTC) 전체 E2E 테스트 실행
   - 주요 기능 변경 시 즉시 트리거

**구현 난이도**: Medium
**예상 임팩트**: High
**영향 파일**: 
- `e2e/*.spec.ts` (신규 4개 파일 추가)
- `e2e/fixtures.ts` (신규)
- `playwright.config.ts` (재시도 로직, 타임아웃 설정)
- `.github/workflows/test.yml` (신규 또는 수정)
- `package.json` (테스트 스크립트 추가)

**예상 작업량**: 14-18시간
