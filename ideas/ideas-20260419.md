### Idea 1: 응답 파싱 안정성 강화 — JSON 구조 검증 계층 도입

**발견 근거**: 
- 최근 7일 커밋 중 JSON 파싱 관련 fix 5건 (라인 4e5a58a, 7dc4bed, c52252b, 3b4e1ee, 97048ac)
- "JSON parse failure when Claude appends trailing prose after closing brace" (3b4e1ee)
- "ensure all required fields exist after Claude response parse" (c52252b)
- `lib/response-parser.ts` 핫스팟 파일 (2회 수정)

**현재 문제**: 
Claude API 응답이 불완전하거나 추가 텍스트를 포함할 때 JSON 파싱 실패 → 사용자 분석 결과 손실 또는 부분 손실 → 재분석 요청 필요로 UX 마찰 발생

**제안**: 
1. 응답 파싱 전 **스트림 정규화 전처리기** 추가 (닫는 괄호 이후 텍스트 제거, 불완전한 구조 감지)
2. **Zod 스키마 기반 검증** → 누락된 필드 자동 감지 및 기본값 삽입
3. **파싱 실패 시 재시도 로직** → 동일 요청에 대해 최대 2회 재분석 자동 실행
4. 파싱 성공률을 데이터베이스에 기록 → 모니터링 대시보드에서 추이 추적

```typescript
// lib/response-parser.ts 개선 예시
export const parseClaudeResponse = async (raw: string) => {
  // 1. 정규화
  const normalized = normalizeJsonString(raw);
  
  // 2. Zod 검증 + 자동 기본값 삽입
  const validated = AnalysisSchema.safeParse(normalized);
  
  if (!validated.success) {
    // 3. 재시도 로직
    return retryClaudeAnalysis();
  }
};
```

**구현 난이도**: Medium
**예상 임팩트**: High (반복되는 에러 제거 → 분석 성공률 향상)
**영향 파일**: `lib/response-parser.ts`, `lib/claude.ts`, `app/api/analyze/route.ts`, `app/api/analyze-flow/route.ts`
**예상 작업량**: 2-3일

### Idea 3: TypeScript 빌드 안정성 강화 — E2E 테스트 타입 에러 제거 및 CI/CD 신뢰도 향상

**발견 근거**: 
- 최신 QA 리포트: "타입 체크 ❌ FAIL" (빌드 ⚠️ FAIL)
- CRITICAL 이슈: `@playwright/test` 모듈 해석 실패 (TS2307)
- HIGH 이슈: `e2e/settings.spec.ts` 암묵적 `any` 타입 (TS7031)
- E2E 테스트 3개 파일 모두 타입 에러 발생 → 런타임 에러 위험

**현재 문제**: 
- `tsconfig.json`과 `tsconfig.e2e.json` 간 설정 불일치
- 로컬 개발과 CI/CD 환경 간 테스트 결과 차이 가능성
- IDE 자동완성 미지원 → 개발 속도 저하

**제안**: 
1. **`tsconfig.json` 통합 설정**
   ```json
   {
     "compilerOptions": {
       "skipLibCheck": true,
       "moduleResolution": "node",
       "resolveJsonModule": true,
       "types": ["@playwright/test", "node", "@types/jest"]
     }
   }
   ```

2. **E2E 테스트 기본 픽스처 생성** → `e2e/fixtures.ts`
   ```typescript
   import { test as base, Page } from '@playwright/test';
   
   type TestFixtures = {
     authenticatedPage: Page;
   };
   
   export const test = base.extend<TestFixtures>({
     authenticatedPage: async ({ page }, use) => {
       // 인증 로직
       await use(page);
     }
   });
   ```

3. **npm script 추가**
   ```json
   "type-check:e2e": "tsc --noEmit -p tsconfig.e2e.json"
   ```
   → CI 파이프라인에 통합: `npm run type-check && npm run type-check:e2e`

4. **E2E 테스트 파일 타입 어노테이션 추가**
   ```typescript
   // e2e/settings.spec.ts
   import { test, expect, Page } from '@playwright/test';
   
   test('description', async ({ page }: { page: Page }) => {
     // ...
   });
   ```

**구현 난이도**: Low
**예상 임팩트**: Medium (빌드 성공률 100% 달성, CI/CD 신뢰도 향상, 개발 생산성 ↑)
**영향 파일**: `tsconfig.json`, `tsconfig.e2e.json`, `playwright.config.ts`, `e2e/*.spec.ts`, `.github/workflows/` (CI 설정)
**예상 작업량**: 1-2일
