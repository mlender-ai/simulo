## Idea 1: Response Parser 안전성 강화 — JSON 파싱 오류 근본 해결

**발현 근거**: 
- Git Log에서 JSON 파싱 관련 fix 커밋 5개 반복 (fix: JSON parse failure from truncation, fix: JSON parse failures from truncation, fix: prevent localStorage quota exceeded, fix: ensure all required fields exist, fix: guard against undefined issues array crash)
- `lib/response-parser.ts` 핫스팟 파일 (3회 변경)
- QA 리포트에서 "Claude appends trailing prose after closing brace" 패턴 명시

**현재 문제**: 
- Claude API 응답이 max_tokens 제한으로 JSON 완성 전 중단 → 파싱 실패
- 응답 뒤에 마크다운 텍스트 추가 → 정규식 기반 추출 실패
- undefined 배열 접근 → 런타임 충돌 (ReportTabs, ComparisonReportTabs)
- localStorage 용량 초과 → 캐시 손상

**제안**: 
1. **Robust JSON Extraction Layer 구축**
   - 응답에서 마지막 완전한 JSON 블록만 추출하는 안전한 파서 작성
   - 불완전한 JSON에 대해 재시도 로직 추가 (exponential backoff)
   - 응답 전에 max_tokens를 동적으로 조정

2. **3-Tier Validation Pipeline 도입**
   ```typescript
   // lib/response-parser.ts 리팩토링
   1. Zod 스키마 기반 정적 검증
   2. 필드별 null coalescing 처리 (issues[], risks[] 기본값 [])
   3. 저장 전 용량 체크 후 IndexedDB 자동 전환
   ```

3. **개발자 경험 개선**
   - 파싱 실패 시 에러 로그에 원본 응답의 처음/끝 100자 포함
   - 개발 모드에서 파싱 결과를 시각화하는 디버그 패널 추가

**구현 난이도**: Medium
**예상 임팩트**: High (40% fix 커밋 감소 기대)
**영향 파일**: 
- `lib/response-parser.ts`
- `lib/claude.ts`
- `components/ReportTabs.tsx`
- `components/ComparisonReportTabs.tsx`
- `app/api/analyze/route.ts`

**예상 작업량**: 2-3일

## Idea 3: TypeScript 빌드 파이프라인 안정화 — E2E 테스트 타입 검증 자동화

**발현 근거**: 
- QA 리포트 CRITICAL: "TypeScript 모듈 해석 실패" — `@playwright/test` 미등재
- 빌드 상태: ⚠️ FAIL, 타입 체크: ❌ FAIL
- E2E 테스트 3개 파일(`api.spec.ts`, `pages.spec.ts`, `settings.spec.ts`)에서 TS7031 에러 (암묵적 any)
- package.json의 devDependencies에 `@playwright/test` 존재하지만 타입 선언 미흡

**현재 문제**: 
- `tsconfig.json`이 Playwright 타입을 인식하지 못함 → IDE 자동완성 불가, CI에서 빌드 실패
- E2E 테스트 파라미터가 타입 없음 (`async ({ page })`) → 런타임 에러 가능성 높음
- CI/CD와 로컬 개발 환경에서 빌드 결과 불일치
- `npm run type-check:e2e` 스크립트 있지만 CI에 통합되지 않음

**제안**: 
1. **TypeScript 설정 통합**
   ```json
   // tsconfig.json 수정
   {
     "compilerOptions": {
       "skipLibCheck": true,
       "moduleResolution": "node",
       "types": ["@playwright/test", "node"],
       "resolveJsonModule": true,
       "esModuleInterop": true
     }
   }
   ```

2. **E2E 테스트 기본 설정 표준화**
   ```typescript
   // e2e/fixtures.ts (신규)
   import { test as base, expect } from '@playwright/test';
   
   export const test = base;
   export { expect };
   ```
   - 모든 E2E 테스트가 이를 import하도록 통일

3. **package.json 검증 추가**
   ```bash
   "validate": "npm run type-check && npm run type-check:e2e && npm run lint && npm run build"
   ```
   - `npm run validate`가 모든 타입 체크 통과 확인

4. **GitHub Actions에 E2E 타입 체크 통합**
   ```yaml
   - name: Type check E2E tests
     run: npm run type-check:e2e
   ```
   - CI 파이프라인에서 빌드 전에 타입 검증

**구현 난이도**: Low
**예상 임팩트**: Medium (빌드 실패율 0%, 개발자 경험 향상, CI 신뢰성 증대)
**영향 파일**: 
- `tsconfig.json`
- `tsconfig.e2e.json`
- `playwright.config.ts`
- `e2e/*.spec.ts` (3개 파일)
- `.github/workflows/` (CI 설정)
- `package.json`

**예상 작업량**: 0.5-1일
