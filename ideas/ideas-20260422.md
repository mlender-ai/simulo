### Idea 1: API 응답 캐싱 + 요청 큐잉 레이어 도입

**발견 근거**: 
- Git 로그에서 `770c8c0 fix: add exponential backoff retry to all Claude API calls` 커밋 확인
- `app/api/analyze/route.ts`가 최근 7일 변경 사항의 21% 차지 (4/19 핫스팟)
- 동일한 이미지/코드에 대한 중복 분석 요청이 빈번할 가능성

**현재 문제**:
- Exponential backoff만으로는 API 레이트 제한(Rate Limit)과 높은 응답 지연에 대응 불충분
- 동일 입력에 대한 반복 분석 요청이 Claude API 비용 증가 및 사용자 경험 악화 초래
- 분석 요청 폭주 시 순차 처리로 인한 타임아웃 위험

**제안**:
1. **응답 캐시 레이어**: 입력 파일(hash) × 분석 모드별 Redis/in-memory 캐시 (24시간 TTL)
   - OCR 분석, 코드-Figma 분석, 흐름 분석 결과를 각각 캐싱
2. **요청 큐잉**: Bull/BullMQ를 활용한 비동기 작업 큐
   - 즉각적 응답 필요 분석(OCR)은 우선순위 높음
   - 배경 분석(개선안 생성)은 우선순위 낮음으로 설정
3. **재사용 표시**: UI에서 "캐시됨" 배지 표시로 사용자 신뢰도 향상

**구현 난이도**: Medium
**예상 임팩트**: High (API 비용 20-30% 절감, 사용자 대기 시간 50% 단축)
**영향 파일**: 
- `app/api/analyze/route.ts`
- `app/api/analyze-flow/route.ts`
- `app/api/code-to-figma/route.ts`
- `lib/claude.ts` (새 캐시 유틸리티 함수)

**예상 작업량**: 3-4일

### Idea 3: E2E 테스트 타입 안정성 확보 + CI/CD 파이프라인 강화

**발견 근거**:
- QA 리포트에서 `playwright.config.ts`, `e2e/settings.spec.ts` 모두 TS2307/TS7031 타입 에러
- 최근 커밋: `npm install --save-dev @playwright/test@latest` 관련 패키지 업데이트
- 빌드 실패(⚠️ FAIL), 타입체크 실패(❌ FAIL) 상태 지속
- 3개 E2E 스펙 파일 모두 영향받음 (`api.spec.ts`, `pages.spec.ts`, `settings.spec.ts`)

**현재 문제**:
1. **모듈 해석 실패**: `@playwright/test` 타입 선언 누락 → IDE 자동완성 불가능
2. **암묵적 any 타입**: 테스트 함수 매개변수 `page` 타입 미정의 → 런타입 에러 위험
3. **빌드 파이프라인 차단**: TypeScript 타입 체크 실패로 배포 불가능
4. **개발자 경험 저하**: 타입 안정성 부족으로 테스트 작성 시간 증가

**제안**:
1. **Playwright 설정 정규화**
   ```typescript
   // tsconfig.json 수정
   {
     "compilerOptions": {
       "skipLibCheck": true,
       "moduleResolution": "node",
       "types": ["@playwright/test", "node"]
     }
   }
   ```

2. **E2E 테스트 기본 구조 표준화**
   ```typescript
   // e2e/fixtures.ts (신규)
   import { test as base, expect, type Page } from '@playwright/test';
   
   type TestFixtures = {
     authenticatedPage: Page;
     apiContext: APIRequestContext;
   };
   
   export const test = base.extend<TestFixtures>({
     authenticatedPage: async ({ page }, use) => {
       await page.goto('/login');
       await page.fill('[data-testid=email]', 'test@example.com');
       await use(page);
     },
   });
   ```

3. **모든 E2E 스펙에 타입 지정 강제**
   - `e2e/**/*.spec.ts`에서 `({ page }: { page: Page })` 형식 적용
   - Pre-commit 훅으로 타입 검증

4. **CI/CD 강화**
   - GitHub Actions에 `npm run type-check:e2e` 단계 추가
   - 빌드 실패 시 자동 PR 댓글 생성 (원인 + 해결 방법 제시)

**구현 난이도**: Low
**예상 임팩트**: High (배포 차단 해소, 테스트 신뢰성 확보)
**영향 파일**: 
- `tsconfig.json`
- `tsconfig.e2e.json` (신규 또는 수정)
- `playwright.config.ts`
- `e2e/api.spec.ts`
- `e2e/pages.spec.ts`
- `e2e/settings.spec.ts`
- `.github/workflows/*.yml` (CI/CD 파이프라인)

**예상 작업량**: 1-2일
