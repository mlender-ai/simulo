# Product Ideas — 2026-04-22

## 분석 요약
- 분석 기간: 최근 7일
- 핵심 발견사항:
  1. **기능 확장 가속화** (코드→Figma 역방향 분석, 3-Pass OCR 파이프라인, 영상 업로드+프레임 분석 등 고도화 기능 연속 추가)에 비해 **신규 기능 간 통합 UX 흐름이 부재** — 기능별로 독립된 페이지가 늘어나는 구조적 분산 발생
  2. **`app/api/analyze/route.ts`가 핫스팟 1위(4회 변경)** 임에도 단일 파일로 유지 — 다양한 입력 타입(이미지, 영상, OCR, 코드)을 하나의 라우트에서 처리하는 분기 복잡도 위험
  3. **OCRReviewModal, MediaUploader, HeatmapViewer** 등 신규 컴포넌트가 추가되었지만, 이를 활용한 **분석 결과의 편집·수정 워크플로우**가 미완성 상태로 추정

### Idea 1: 분석 입력 타입별 라우트 핸들러 분리 — `analyze/route.ts` 책임 분산

**발견 근거**:
- 핫스팟 파일 Top 1: `app/api/analyze/route.ts` — 7일간 4회 변경 (전체 파일 중 최다)
- 최근 커밋에서 이미지, 영상(`upload-video`), OCR(`ocr-extract`), 코드(`code-to-figma`) 분석이 모두 단기간에 추가됨
- 기존 아이디어(04-19~04-21)에서 다루지 않은 **API 레이어 설계 문제** (이전 아이디어는 claude.ts 분리, 재시도 로직, 파싱 계층에 집중)

**현재 문제**:
- `analyze/route.ts` 하나에 입력 타입(이미지/영상/OCR/코드) 분기 로직이 혼재할 가능성 — 새 입력 타입 추가 시마다 동일 파일을 수정해야 하는 구조
- 단일 파일 변경이 잦을수록 무관한 분석 타입에 회귀 버그를 유발할 위험 증가
- `analyze-flow/route.ts`, `ocr-extract/route.ts`, `code-to-figma/route.ts`가 이미 분리되어 있음에도 `analyze/route.ts`가 계속 변경되는 것은 중앙 집중 분기 패턴이 유지되고 있다는 신호

**제안**:
```
app/api/analyze/
├── route.ts              ← 요청 라우팅만 담당 (입력 타입 판별 후 위임)
├── handlers/
│   ├── image.handler.ts  ← 이미지 단일 분석
│   ├── video.handler.ts  ← 영상 프레임 분석
│   ├── ocr.handler.ts    ← OCR 텍스트 기반 분석
│   └── comparison.handler.ts ← 비교 분석
└── middleware/
    └── validate-input.ts ← 공통 입력 검증
```
- `route.ts`는 `Content-Type` 또는 요청 바디의 `type` 필드를 보고 적절한 핸들러로 위임만 수행
- 각 핸들러는 독립 테스트 가능한 순수 함수 형태로 작성
- 핸들러 추가 시 기존 핸들러를 건드리지 않아도 됨 (OCP 준수)

**구현 난이도**: Medium
**예상 임팩트**: High
**영향 파일**:
- `app/api/analyze/route.ts`
- `app/api/upload-video/route.ts` (참조 패턴 정렬)
- `app/api/ocr-extract/route.ts` (참조 패턴 정렬)
- `lib/claude.ts` (핸들러에서 직접 호출 구조로 연결)

**예상 작업량**: 1.5일

### Idea 2: OCR 보정 결과 인라인 편집 — `OCRReviewModal` 워크플로우 완성

**발견 근거**:
- 커밋 `2400afd`에서 3-Pass OCR 보정 파이프라인 신설, `components/OCRReviewModal.tsx` 컴포넌트 존재 확인
- 그러나 OCR 결과를 **사용자가 검토하고 수정한 뒤 분석에 반영하는 end-to-end 플로우**가 커밋 히스토리상 완결된 흔적 없음
- 기존 아이디어(04-19~04-21)는 OCR을 다루지 않음 — 완전히 새로운 관점

**현재 문제**:
- 3-Pass OCR이 아무리 정교해도 손글씨·저해상도 UI 이미지에서는 오인식 발생 불가피
- `OCRReviewModal`이 존재하지만 모달 내에서 텍스트를 **직접 수정 → 재분석 트리거**하는 인터랙션이 없다면 사용자는 잘못된 OCR 결과를 그대로 분석에 넘기게 됨
- UX 마찰: "OCR이 틀렸는데 고칠 방법이 없다" → 분석 결과 신뢰도 저하 → 툴 자체 불신으로 이어짐

**제안**:
```
OCRReviewModal 기능 확장:
1. [인라인 편집] 인식된 텍스트 블록을 클릭하면 바로 수정 가능한 contenteditable 영역 제공
2. [신뢰도 표시] 각 텍스트 블록에 OCR 신뢰도 점수(%) 색상 표시 (빨강 < 70%, 노랑 70~90%, 초록 > 90%)
3. [부분 재시도] 특정 블록만 선택해 OCR 재인식 요청 가능 (전체 재실행 대신)
4. [확정 후 분석] "이 텍스트로 분석 시작" 버튼 → 수정된 텍스트를 분석 파이프라인에 주입
```
- `api/ocr-extract/route.ts`에 `partial: true` + `region` 파라미터 추가로 부분 재시도 지원
- 수정된 OCR 텍스트는 `history` 테이블에 `ocr_corrected_text` 컬럼으로 저장 → 향후 OCR 모델 개선 학습 데이터로 활용 가능

**구현 난이도**: Medium
**예상 임팩트**: High
**영향 파일**:
- `components/OCRReviewModal.tsx`
- `app/api/ocr-extract/route.ts`
- `app/api/analyze/route.ts` (수정된 텍스트 수신 처리)
- `app/analyze/page.tsx` (모달 → 분석 트리거 연결)

**예상 작업량**: 2일

### Idea 3: 영상 프레임 분석 결과의 타임라인 뷰 — 비선형 UX 흐름 가시화

**발견 근거**:
- 커밋 `39781f4`에서 영상 업로드 + 프레임 분석 기능 추가 — Simulo의 입력 범위가 정적 이미지에서 동적 영상으로 확장된 가장 최신 기능
- `components/HeatmapViewer.tsx` 존재 — 현재는 단일 이미지 기준 히트맵으로 추정되나, 영상에서 추출된 다수 프레임에 대한 **시간축 기반 시각화**는 별도 컴포넌트 없음
- 기존 아이디어(04-19~04-21) 중 영상 분석 관련 아이디어 전무 — 가장 새로운 기능에 대한 첫 UX 개선 제안

**현재 문제**:
- 영상에서 추출한 N개 프레임의 분석 결과가 현재 어떻게 표시되는지 명확하지 않으나, 기존 `ReportTabs` 구조에 끼워 넣으면 **"프레임 3에서 사용자가 혼란을 겪음"** 같은 시간적 맥락이 소실됨
- 사용자 테스트 영상에서 중요한 것은 **"어느 시점에, 어떤 화면에서, 어떤 문제가 발생했는가"** — 이를 타임라인 없이 표현하면 분석 가치가 절반 이하로 떨어짐
- UX 마찰: 프레임별 분석 결과를 스크롤로 순서대로 보는 것과, 타임라인에서 문제 구간을 직접 클릭해 점프하는 것은 인지 부하 면에서 크게 다름

**제안**:
```
VideoAnalysisTimeline 컴포넌트 신설:

[타임라인 바]
0s ──●──────────●───────●──── 끝
     ↑           ↑       ↑
   프레임1     프레임5  프레임8
  (정상)      (⚠️마찰)  (❌오류)

기능:
1. [프레임 썸네일 스트립] 추출된 프레임을 시간순으로 가로 배열
2. [이슈 마커] 분석 결과에서 CRITICAL/HIGH 이슈가 발견된 프레임에 색상 마커 표시
3. [클릭 → 상세] 프레임 클릭 시 해당 프레임의 분석 결과 패널이 우측에 표시
4. [이슈 구간 요약] 전체 영상에서 "문제 밀집 구간" 자동 하이라이트
5. [PDF 내보내기 연동] 타임라인 스냅샷 + 이슈 구간 요약을 export에 포함
```
- `app/report/[id]/page.tsx`에서 입력 타입이 `video`인 경우 `VideoAnalysisTimeline`으로 자동 전환
- `ShareExportPanel`에 영상 분석 전용 export 포맷 추가 (`타임라인 + 프레임별 이슈` PDF)

**구현 난이도**: High
**예상 임팩트**: High
**영향 파일**:
- `components/VideoAnalysisTimeline.tsx` (신규)
- `app/report/[id]/page.tsx`
- `app/api/upload-video/route.ts` (프레임 메타데이터 타임스탬프 포함 여부 확인)
- `components/ShareExportPanel.tsx`
- `app/api/export/pdf/[id]/route.ts`

**예상 작업량**: 3일
